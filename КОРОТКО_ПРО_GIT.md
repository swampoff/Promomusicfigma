# 🎯 GIT ДЛЯ ЧАЙНИКОВ - ЗА 2 МИНУТЫ

## Ты спросил: "как файлы накладываются друг на друга в git?"

**Ответ: НИКАК!** Git работает по-другому! 😊

---

## 📚 Git = Умная книга истории

### Не копирование, а дневник изменений!

```
┌─────────────────────────────────────┐
│ БЫЛО (вчера):                       │
│ const name = "Вася";                │
└─────────────────────────────────────┘
          ↓
┌─────────────────────────────────────┐
│ СТАЛО (сегодня):                    │
│ const name = "Петя";                │
│ const age = 25;                     │
└─────────────────────────────────────┘
          ↓
┌─────────────────────────────────────┐
│ GIT ЗАПОМНИЛ:                       │
│ - Строка 1: "Вася" → "Петя"        │
│ + Строка 2: добавлена "const age"  │
└─────────────────────────────────────┘
```

**Git НЕ удаляет старое!** Он просто записывает: "вот что изменилось"

---

## 🔄 Что происходит когда ты копируешь файлы:

### Сценарий:

```bash
# У тебя на Desktop:
App.tsx  (100 строк, версия A)

# Ты работал в Figma Make:
App.tsx  (120 строк, версия B)

# Скопировал файлы из Figma Make → Desktop
# Теперь на Desktop:
App.tsx  (120 строк, версия B)
```

---

### Git увидит:

```bash
git status

# Вывод:
modified: App.tsx
  20 строк добавлено
  0 строк удалено
```

**Git понимает:** "О, App.tsx изменился - добавили 20 строк!"

---

### Git НЕ делает:

❌ "Удалю старый файл полностью"  
❌ "Наложу новый файл сверху"  
❌ "Потеряю историю"

---

### Git делает:

✅ Сравнивает построчно  
✅ Сохраняет только изменения  
✅ Запоминает историю  
✅ Можешь вернуться к любой версии

---

## 📊 Пример реальный:

### У тебя было:

```tsx
// App.tsx (версия на GitHub)
function App() {
  return <div>Hello</div>
}
```

### Ты изменил в Figma Make:

```tsx
// App.tsx (версия в Figma Make)
function App() {
  return (
    <div>
      <h1>Hello World!</h1>
      <button>Click me</button>
    </div>
  )
}
```

### Скопировал на Desktop и сделал git push:

Git отправит на GitHub:

```diff
function App() {
-  return <div>Hello</div>
+  return (
+    <div>
+      <h1>Hello World!</h1>
+      <button>Click me</button>
+    </div>
+  )
}
```

**Только изменения!** Не весь файл!

---

## 💡 Почему это безопасно:

### 1. Git хранит ВСЮ историю

```bash
git log

# Вывод:
commit abc123 - "feat: добавил кнопку" (сегодня)
commit def456 - "feat: начальная версия" (вчера)
```

---

### 2. Можешь вернуться к любой версии

```bash
# Вернуться к вчерашней версии
git reset --hard def456

# Вернуться к сегодняшней
git reset --hard abc123
```

---

### 3. Git видит конфликты

Если кто-то ЕЩЕ изменил тот же файл:

```bash
git pull

# Git скажет:
CONFLICT in App.tsx
Auto-merging failed
```

И покажет ЧТО конфликтует:

```tsx
<<<<<<< HEAD (твоя версия)
<h1>Hello World!</h1>
=======
<h1>Hi there!</h1>
>>>>>>> remote (версия на GitHub)
```

Ты выбираешь что оставить!

---

## 🎯 ГЛАВНОЕ:

### Git НЕ "накладывает файлы друг на друга"!

Git работает как **умный редактор**:

1. **Сравнивает** старую и новую версию
2. **Находит** что изменилось
3. **Сохраняет** только изменения
4. **Хранит** всю историю

---

## 🆘 Что если я скопирую файлы 100 раз?

**Ничего страшного!**

```bash
# Git каждый раз просто увидит:
git status

# Вывод:
modified: 5 files
123 строки добавлено
45 строк удалено
```

**И сохранит изменения!** Не "наложит" файлы!

---

## 📋 Команды для проверки:

```bash
# Что изменилось?
git status

# Показать изменения подробно (построчно)
git diff

# История изменений
git log --oneline

# Кто что изменил в файле
git blame App.tsx
```

---

## 🎯 РЕЗЮМЕ:

**Git = дневник изменений, а НЕ копирование файлов!**

Поэтому:
- ✅ Безопасно копировать файлы
- ✅ Git сам найдет что изменилось
- ✅ Можешь вернуться к любой версии
- ✅ История не потеряется

**НО:**
- ❌ Копировать каждый раз = геморрой!
- ✅ Лучше работай прямо из Desktop папки!
